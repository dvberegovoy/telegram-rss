# telegram-rss

Превращает публичные Telegram-каналы в RSS-ленты. Всё работает на GitHub бесплатно:
GitHub Actions по будням в 08:30 МСК парсит публичные веб-превью каналов
(`https://t.me/s/<канал>`), собирает RSS и публикует их через GitHub Pages.

Поскольку сборка идёт в облаке GitHub, локальная блокировка Telegram значения не
имеет — подписчик (например, агент) забирает готовый RSS с `github.io`.

## Настройка

1. **Создай репозиторий** (публичный — тогда минуты GitHub Actions безлимитны;
   приватный тоже подойдёт, расход ~40 мин/мес, это в пределах бесплатного лимита).
   Загрузи в него все файлы из этого набора.

2. **Укажи каналы** в `channels.txt` — по одному на строку. Можно писать
   `@name`, `name` или ссылку. Для первого запуска оставлены `telegram` и `durov`,
   чтобы проверить работу; потом удали их.

3. **Включи GitHub Pages:** Settings → Pages → Source → **Deploy from a branch** →
   ветка `main`, папка `/docs` → Save.

4. **Проверь сборку:** вкладка Actions → workflow «Build Telegram RSS» →
   **Run workflow**. После завершения в `docs/feeds/` появятся `.xml`, а по адресу
   `https://<логин>.github.io/<репозиторий>/` откроется страница со списком лент.

## Адреса лент

- Сводная (все каналы, новые сверху):
  `https://<логин>.github.io/<репозиторий>/feeds/all.xml`
- По конкретному каналу:
  `https://<логин>.github.io/<репозиторий>/feeds/<канал>.xml`

## Расписание

Cron в `.github/workflows/build.yml` стоит на `30 5 * * 1-5` — 05:30 UTC = 08:30 МСК,
по будням. Москва постоянно UTC+3 (перехода на летнее время нет), пересчёт не нужен.

> Планировщик GitHub Actions не пунктуален: при нагрузке запуск может сдвинуться
> на 5–30 минут. Если нужна гарантия свежих данных к 9:00, поставь запуск раньше —
> замени cron на `0 5 * * 1-5` (08:00 МСК).

## Настраиваемые параметры

В начале `telegram_rss.py`:

- `MAX_POSTS` — сколько постов держать в ленте канала (по умолчанию 20).
- `COMBINED_LIMIT` — размер сводной ленты (по умолчанию 100).
- `RETRIES`, `RETRY_DELAY`, `REQUEST_TIMEOUT` — поведение при сетевых ошибках.

## Ограничения

- Работают только **публичные** каналы (с `@username` и включённым превью).
  Для приватных нужен другой механизм — авторизация через user-аккаунт по MTProto
  (Telethon) с хранением сессии в secrets.
- Если канал недоступен или у него отключено превью, он пропускается, а остальные
  собираются как обычно. Статус по каждому каналу виден на стартовой странице и в
  логах Actions.
- Скрипт опирается на текущую вёрстку `t.me/s/`. Если Telegram изменит HTML-классы,
  ленты окажутся пустыми («нет постов» в логах) — тогда нужно поправить CSS-селекторы
  в `parse_posts()`.

## Подключение к Hermes

Подпиши агента на `all.xml` для общей картины или на отдельные `<канал>.xml`
для гранулярности. Каждый элемент ленты содержит полный текст поста (в
`description` — текстом, в `content:encoded` — с разметкой и картинками),
прямую ссылку на пост и дату публикации.
